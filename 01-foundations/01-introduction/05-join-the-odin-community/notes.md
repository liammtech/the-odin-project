# JOIN THE ODIN COMMUNITY

## Introduction
Working and collaborating with other people is an important part or working as a web developer. Therefore, The Odin Project encourages you to participate in the online chat
community.

By joining the community, you can grow alongside other Odinites and help each other learn
web development.

## Lesson overview
- Learn about The Odin's community and how to join it
- Explain how to ask good and detailed questions
- Explain good practices for helping others with their questions

## Why a community is awesome for you
- Learning web development will be a long and arduous journey
- You can make it more fun by joining the Odin community
- No matter your pace, there will be someone further ahead willing to help
- Similarly, helping others a few steps behind you will help your learning stick

## Why a community is awesome for Odin
- TOP works hard to update existing lessons and produce new content
- They are open to feedback

## Before asking for help
- Remember there is always a community to turn to for help
- You don't need to know how to solve every problem straight away, BUT:
- You do need to have a general idea of where you are going
- Sometimes the problem is your approach, not your code

If you're feeling stuck, it's a good time to pause and take a breather. Break the problem
down into little pieces and then decide what is really holding you back. This technique is
called "Rubber Duck Debugging":

https://en.wikipedia.org/wiki/Rubber_duck_debugging [ X ]

Also, read about how to use Google to solve your programming questions:

https://web.archive.org/web/20250918082145/https://old.codinginflow.com/google-programming-questions [ X ]

If these methods don't yield a result, it's time to turn to the community.

## Asking for help
- First thing to keep in mind: don't ask to ask
- When asking, put together a great question
- Include context:
    - What do you think the problem is?
    - What exactly do you want to happen?
    - What is actually happening?
    - How did you get there?
    - What have you tried so far?
- If you can't pintpoint the problem, you can share a screenshot (especially useful for
showing command line output)
- However you should always include the corresponding code creatting the error

## Formatting your questionsdd
Asking your questions in a readable format helps everyone debug them better. Here are some
ways to go about that:

- If you're having trouble on the command line, make sure to include both your input and 
the error message you're getting

- In the chat rooms, code can be formatted using backticks (``)

- One line of code: one backtick before and after your code:

    `your_code()`

- Multiple lines of code: three above and three below:

    ```
    your_code1()
    your_code2()
    ```

- Can also use code highlighting specific to the language:

    ```js or javascript
    your_code1()
    your_code2()
    ```
    Even works in markdown - neat!

## How to help others solve coding problems

As well as asking good questions, it's important to know how to help others effectively.

1. Instead of answering the question, guide them to the answer

2. Help only when you are certain of the answer

3. Help only when no one else is currently helping

4. Help only when you have enough/plenty of time

5. Adjust your expectations to their level

6. Ask for clarifications

7. Ask for live code (e.g. a CodePen)

8. Do not answer Googleable questions

9. Do not answer questions covered in our curriculum

10. Answer the question before pointing out other problems

11. Encourage students to use a debugger

12. Watch out for students that need to take a step back

13. Watch out for students that are in over their head

14. Admit when the problem goes beyond your current knowledge

15. Be patient

16. Duck out of the conversation if you get frustrated
